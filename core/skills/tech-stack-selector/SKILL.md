# Tech Stack Selector

プロジェクト開始時に、ユーザーの要望から最適なプラットフォーム×フレームワークを提案する。

---

## 🎯 使用タイミング

| シナリオ | 必要性 |
|---------|--------|
| 「Flutterでアプリ作って」 | **不要**（フレームワーク明示） |
| 「モバイルアプリ作って」 | **必要**（フレームワーク未定） |
| 「アプリ作って」 | **必要**（プラットフォームも未定） |

---

## 📋 選定フロー

### Step 1: 要件ヒアリング

ユーザーに以下を確認:

```
1. どのプラットフォームで動かしたいですか？
   □ スマートフォン（iOS/Android）
   □ Webブラウザ
   □ デスクトップ（Windows/Mac/Linux）
   □ 複数対応したい

2. チームの技術スタック経験は？
   □ JavaScript/TypeScript
   □ Dart
   □ 特になし / 新規学習OK

3. 特に重視する点は？
   □ 開発速度（とにかく早くリリース）
   □ パフォーマンス（ネイティブ級の速度）
   □ OTA更新（ストア審査なしで更新）
   □ 既存Webとの統合
```

---

### Step 2: 選択肢提示

要件に基づき、**関連する選択肢のみ**を提示:

#### モバイルアプリの場合

| フレームワーク | 言語 | 特徴 | 推奨ケース |
|--------------|------|------|-----------|
| **Flutter** | Dart | 高パフォーマンス、美しいUI | パフォーマンス重視、カスタムUI |
| **Expo** | TypeScript | 開発速度最速、OTA更新可能 | 素早くリリース、JS/TS経験あり |
| **React Native** | TypeScript | ネイティブ連携柔軟 | 既存RNコード、ネイティブモジュール必要 |

#### Webアプリの場合

| フレームワーク | 言語 | 特徴 | 推奨ケース |
|--------------|------|------|-----------|
| **Next.js** | TypeScript | SSR/SSG、React | SEO重視、大規模 |
| **Nuxt** | TypeScript | Vue.js | Vue経験者 |
| **SvelteKit** | TypeScript | 高速、軽量 | パフォーマンス重視 |

#### デスクトップアプリの場合

| フレームワーク | 言語 | 特徴 | 推奨ケース |
|--------------|------|------|-----------|
| **Electron** | TypeScript | Web技術でデスクトップ | 既存Webを移植 |
| **Tauri** | Rust+TS | 軽量、高速 | パフォーマンス重視 |
| **Flutter Desktop** | Dart | クロスプラットフォーム | モバイルと共有 |

#### 複数プラットフォーム対応

| 組み合わせ | 対応 | 特徴 |
|-----------|------|------|
| **Flutter** | Mobile + Web + Desktop | 1コードベース |
| **Expo + Next.js** | Mobile + Web | 共通言語（TS） |
| **React Native + Electron** | Mobile + Desktop | 共通言語（TS） |

---

## 🔄 決定後のフロー

ユーザーがフレームワークを選択したら:

```
1. 選択されたフレームワークのスキルを取得
   例: Flutter選択 → flutter-development, ai-flutter-guidelines 取得

2. 該当ドメインの開発フローを開始
```

| React Native | ★★ | ★★ | ★★ |

| ネイティブ(Swift/Kotlin) | ★★★ | ★★★ | ★★★ |

| フレームワーク | Hot Reload | デバッグ | ドキュメント |
|--------------|-----------|---------|-------------|
| Flutter | ★★★ | ★★★ | ★★★ |
| Expo | ★★★ | ★★★ | ★★★ |
| React Native | ★★ | ★★ | ★★ |

### エコシステム

## 出力テンプレート

選定完了時、以下を出力:

```markdown
## 技術スタック選定結果

**プラットフォーム**: Mobile (iOS/Android)
**フレームワーク**: Flutter
**言語**: Dart

### 選定理由
- カスタムUIの要件に最適
- 高パフォーマンスが必要
- 開発チームはDart学習可能

### 次のステップ
以下のスキルを取得して開発を開始:
- `ai-flutter-guidelines` - 必須ガイドライン
- `flutter-development` - 開発スキル
- `flutter-environment-check` - 環境診断
```
