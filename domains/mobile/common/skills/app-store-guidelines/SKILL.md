# アプリストアガイドライン

App Store（iOS）およびGoogle Play（Android）の審査対応。

---

## 審査リジェクト防止チェックリスト

### 共通

- [ ] プライバシーポリシーをアプリ内およびストアページに記載
- [ ] 必要な権限のみ要求（過剰な権限はNG）
- [ ] 権限要求時に理由を説明
- [ ] クラッシュ・フリーズがない
- [ ] オフライン時の動作が適切

### App Store（iOS）固有

- [ ] ログインが必要な場合はテストアカウント提供
- [ ] 課金はIn-App Purchaseを使用
- [ ] 「beta」「test」などの文言を含まない
- [ ] App Tracking Transparency対応（IDFA使用時）
- [ ] プライバシー栄養ラベル（Privacy Nutrition Labels）設定

### Google Play固有

- [ ] targetSdkVersionが現行要件を満たす
- [ ] 64bit対応
- [ ] Data Safety Form設定
- [ ] アプリ内課金はPlay Billing使用

---

## リジェクト理由トップ

### App Store

| 順位 | 理由 | 対策 |
|------|------|------|
| 1 | バグ・クラッシュ | 十分なテスト |
| 2 | ガイドライン違反 | 最新HIG確認 |
| 3 | プライバシー問題 | 権限要求理由明示 |
| 4 | 外部決済誘導 | IAP使用 |
| 5 | 不完全な機能 | 機能を完成させる |

### Google Play

| 順位 | 理由 | 対策 |
|------|------|------|
| 1 | ポリシー違反 | 最新ポリシー確認 |
| 2 | スパム判定 | オリジナルコンテンツ |
| 3 | Data Safety不備 | 正確に申告 |
| 4 | 機能不全 | 十分なテスト |

## メタデータ

### 必須項目

| 項目 | iOS | Android |
|------|-----|---------|
| アプリ名 | 30文字 | 30文字 |
| 説明 | 4000文字 | 4000文字 |
| スクリーンショット | 必須 | 必須 |
| アイコン | 1024×1024 | 512×512 |

### スクリーンショット

```
推奨:
- 最低3枚（機能紹介）
- 実際のアプリ画面
- テキストオーバーレイで機能説明
```

---

## 権限要求

### 必須: 使用目的の説明

```
❌ 悪い例: 「カメラへのアクセスを許可してください」
✅ 良い例: 「プロフィール写真を撮影するためにカメラを使用します」
```

### iOS Info.plist

```xml
<key>NSCameraUsageDescription</key>
<string>プロフィール写真を撮影するために使用します</string>
```

### Android Manifest

```xml
<uses-permission android:name="android.permission.CAMERA" />
<!-- runtime permission で説明を表示 -->
```

## 課金

### iOS

- 物理商品以外はIn-App Purchase必須
- 外部決済への誘導禁止
- 購入復元機能必須

### Android

- デジタルコンテンツはPlay Billing
- 代替決済（一部国）

---

## プライバシー

### 必須対応

1. **プライバシーポリシー**: URL必須
2. **データ収集の開示**: 何を収集するか明示
3. **第三者共有**: 広告SDK等の共有を開示

### GDPR/CCPA対応

- EU/カリフォルニアユーザー向け同意取得
- データ削除権への対応

---

## 審査申請のコツ

1. **レビューノート**: 特殊な機能は説明を記載
2. **テストアカウント**: ログイン必須の場合は必ず提供
3. **動画デモ**: 複雑な機能は動画で説明
4. **段階的リリース**: 全ユーザー同時ではなく段階的に
